4. 셸사용하기

배시 셸 환경 설정

셸 변수와 환경 변수

  • 사용자 마다 셸 환경을 설정하기 위해서 셸 변수와 환경 변수를 설정함
    • 셸 변수 : 현재 셸에만 적용되고 서브 셸에는 적용되지 않는다. (지역변수)
    • 환경 변수 : 현재 셸뿐만 아니라 서브 셸에도 적용됨 (전역변수)
    • 로그인 셸에 적용되는 개수가 각 셸 + 환경 변수 = 각 셸 적용 변수
    • set : 변수뿐만 아니라, 함수로 정의된 것도 같이 출력
    • env : 환경 변수만 출력하려면 env을 사용한다.
    • 주요 셸 환경변수
      • 주요 셸 환경변수
      • HISTSIZE : 히스토리 저장 크기
      • PATH : 명령을 탐색할 경로
      • HOME : 사용자 홈 디렉터리의 절대 경로
      • PWD : 작업 디렉터리의 절대 경로
      • LANG : 사용하는 언어
      • SHELL : 로그인 셸
      • LOGNAME : 사용자 계정 이름

      -특정 변수 출력하기

      • 변수의 값을 출력할 때는 변수 이름 앞에 특수문자 $를 붙임
      • ex) echo $SHELL 이런식
  • 셸 변수 설정하기 ( 변수 이름과 문자열 사이에 공백이 있으면 안됨)
    • 변수명 = 문자열이다. 기억하자.
    • SOME1=test (변수 이름과 문자열이니까 공백이 있으면 안된다.)
    • 환경 변수 설정하기 (export)
      • 먼저 셸 변수를 정의하고, export 명령을 사용하여 이를 환경 변수로 설정한다.
      • export SOME1
      • env | grep SOME = 환경변수인 SOME이라는 이름을 갖고있는 변수를 찾아라 export SOME1는 환경변수 SOME1을 등록하겠다는 의미이다.
    • 환경 변수를 다시 셸 변수로 바꾸기 (export -n)
  • 5.에일리어스와 히스토리
    • 에일리어스 (alias) : 쉽게 말해 별명을 만든다는 뜻이다.
    • alias를 지우고싶을땐 간단하게 unalias라고 쓰면된다.
    • 에일리어스 (alias)만 입력한다면 alias로 줄인 명령 목록보기가능
    • alias를 사용해서 긴 명령어를 효과적으로 짧게 만들어준다.
    • alias를 사용해서 여러 명령을 연결해서 하나의 명령으로도 만들 수 있다.
    • alias를 사용하여 자주 사용하는 옵션을 포함하여 새로운 이름을 붙여서 사용 가능
    • ※보통 직장 가서 모르는 명령어를 보면 진짜 모르는 명령어거나 alias이다.
    • 실제로 사용하는 명령어중에 rm=’rm -r’으로 지우려고 할 때 지울거냐고 물어보게 만든다.
  • 배시 셸 환경 설정 사용법 예시들임

    TIP = |(\) 는 파이프라고 부른다.

    • set | grep HIST : HIST가 들어간 문자열의 데이터를 모두 찾아준다.
    • env | grep PATH : env내에서 PATH와 관련된 것을 다 찾아준다.
    • SOME1=test1 (SOME1라는 변수를 만들었다.)
    • set | grep SOME
    • env | grep SOME = 안 나옴.
    • export SOME2=test2
    • env | grep SOME = 둘 다 나온다. SOME1,2
    • unset SOME1, unset SOME2 = 변수 SOME1,2를 지운다.
  • 히스토리
    • 히스토리(history) 사용법
      • history는 내가 쓴 명령어의 목록을 순번대로 볼 수 있다.
      • history에서 추가적인 기능 중 한개인데, !1090을 입력하면 1090번에 있는 clear가 실행되고, !cl 이라고 입력하면 history에서 최근 사용한 cl관련 명령어를 실행한다.
      • more .bash_history를 입력하면 여태까지 입력한 모든 history를 알 수 있다.

      프롬프트에서 사용할 수 있는 문자 많이있지만 중요한것만 외우자.

  • 프롬프트 설정 (좀 어렵다)
    • 일단 무조건 알아야하는거 : 복구 방법 (아래가 복구방법)
    • PS1=’($PWD)’ 작업위치 변경
    • PS1=’(\u@\t $PWD)’ u 현재 사용자 t는 타입. 시간
    • MYPROMPT=$PS1 : MYPROMPT 프롬프트에 있는 값을 PS1에 저장한다.
    • PS1=$MYPROMPT : PS1 변수에 원래 프롬프트 설정정보를 가진 변수를 할당한다.
  • 요약

응용 프로그램 →셸→리눅스커널→하드웨어 (응셸커하)

셸 변수는 현재 셸만 적용

환경 변수는 전체 셸 적용

로그인 셸에 적용되는 개수가 각 셸 + 환경 변수 = 각 셸 적용 변수

set : 변수, 함수 출력

env : 환경 변수만

PWD : 작업 디렉터리의 절대 경로

변수 이름과 문자열이니까 공백이 있으면 안된다.
