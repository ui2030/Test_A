6. 프로세스 관리하기

프로세스 개념
  • 프로세스의 관리 방법
    • 프로세스 관리 → 작업 제어 → 작업 예약
  • 프로세스 : 현재 시스템에서 실행 중인 프로그램
    • 사용자 프로세스, 시스템 프로세스
  • 프로세스의 부모 - 자식 관계
    • 부모 프로세스는 자식 프로세스를 생성하고, 자식 프로세스는 또 다른 자식 프로세스 생성이 가능함
    • 자식 프로세스는 할 일이 끝나면 부모 프로세스에게 결과를 돌려주고 종료된다.
  • 프로세스 번호
    • 각 프로세스는 고유한 번호를 갖고 있다.
    • ex) systemd, init 프로세스는 (PID: 1)번이다. Ktheradd (PID: 2)
    • ex) 참고로 k treadd (PID: 2) 는 k = kernel, d = daemon이라는 뜻이다.
    • ex) systemd는 d = daemon (PID: 1)
  • 특수한 프로세스
    • 데몬(daemon) 프로세스
      • 특정 서비스를 제공하기 위해 존재, 리눅스 커널에 의해 자동 실행되는 경우가 많음.
      • ex) sshd (원격 접속 서비스 데몬), httpd (웹 서비스 데몬), smtpd (메일 전송 데몬) 보통 뒤에 오는 d는 데몬 프로세스이다.
      • DDoS (Distributed Denial of Service)

    • 고아 프로세스
      • 자식 프로세스가 실행 중일 때, 부모 프로세스가 먼저 종료된 자식 프로세스
      • 1번 프로세스(systemd 또는 init)가 새로운 부모 프로세스가 된다.
    • 좀비 프로세스
      • 프로세스가 실행을 종료했는데도, 프로세스 테이블의 목록에 남아 있는 경우.
      • 좀비 프로세스가 프로세스 테이블의 용량이 부족해서 일반 프로세스가 실행되지 않을 수도 있음.
  • 현재 실행 중인 프로세스의 목록을 보는 명령 (ps)
    • 유닉스 옵션(SVR4) : 묶어서 사용이 된다. 붙임표(-)로 시작 (예 : -efpu) [시험엔 이것만 나옴]
    • BSD 옵션 : 묶어서 사용 가능, 붙임표(-)로 시작하지 않는다. (aux)
    • GNU 옵션 : 붙임표 두 개로 시작 (예 : --pid)
    • ps 명령어는 파이프라인으로 grep과 같이 많이 사용됨
      • ps 명령을 옵션 없이 사용하면, 현재 사용자 프로세스에 대한 정보를 출력함
      • ps -e는 시스템에서 실행중인 모든 프로세스 정보 출력
        • PID 1과, 2는 알고 있을 것.
      • ps -f는 UID, PPID, C, STIME 정보를 추가로 출력해준다.

      • 보통 두 개를 조합해서 쓴다.
        • ps -ef | grep kthreadd 처럼..

      • 요약
        • ps : 현재 사용자 프로세스에 대한 정보 출력
        • ps -e : 현재 사용자 프로세스를 포함하여 시스템에서 실행중인 모든 프로세스 정보 출력
        • ps -f : UIDm PPID, C, STIME 정보를 추가로 출력
        • ps -u : user1 사용자가 수행한 프로세스 정보 출력
        • ps -p PID: 특정 PID의 프로세스 정보 출력 (ex : ps -p 1)
        • 참고로 ps -fu는 실행이 되는데 ps -uf는 안됨.

    • 요약에 대해 알았다면 pgrep도 알아보자.
      • pgrep ba는 ba 패턴을 포함하는 모든 프로세스 PID 출력
        • 프로세스의 PID만 알려준다.
      • pgrep -x bash는 정확하게 프로세스의 이름을 안다면
        • 해당 PID를 알려줌
      • pgrep -u user1 -l ba는 특정 사용자(user1)에 대하여 ba 패턴을 포함하는 프로세스의 PID, 명령어를 출력한다.

        PID와 프로세스의 이름을 알려주는 모습.

      • pgrep -l ba는 가장 많이쓰는 명령어 중 하난데, ba와 관련된 PID와 이름을 알려준다.

        PID와 프로세스 이름을 알려줌.

  • 시그널(signal)
    • 프로세스에 간단한 메시지를 알려주는 것.
    • 리눅스에서 지원하는 시그널의 목록은 kill -l 명령어로 알 수 있다.

      SIGINT(2) 에서 INT는 Intfrupt의 준말, (멈춤) Ctrl+c

      SIGKILL(9) 에서 KILL은 그냥 Kill이다. (강제 종료) Ctrl + \ (파이프라인)

      SIGTERM(15) 에서 TERM는 Terminate의 준말. (종료)

      가장 강력한 건 KILL (강제 종료의 의미이다)

      • 명령어를 실행 중 일 때, 명령어를 하나 더 켜서

        ps -ef | grep 00를 치면 00에 해당하는 명령어 PID가 나오는데, 그럴 때 kill -15 PID를 치면된다.

        kill PID를 치면 -15를 생략한 것이다.

        Kill -15로 안되면 -9로 하면된다.

    • pkill 명령을 이용해 프로세스 종료가 가능하다 (다 죽여)
      • 패턴을 나타내는 p의 의미를 갖고있는데.. pkill man을 입력하면 man이 들어간 모든 프로세스가 죽어버린다.
    • killall 명령도 pkill과 유사 기능이지만, 명확한 프로세스 이름을 제시해야한다. (하나만 죽여)

포그라운드 작업(프로세스)
  • 정의 : 사용자가 입력한 명령이 실행되어 결과가 출력될 때까지 기다려야 하는 방식으로 처리되는 프로세스

    또는 입력 후 → 완료시 까지 다른 명령어를 입력하지못함.

  • 백그라운드 작업
    • 명령을 기다리지않고 다른 명령을 실행할 수 있게됨.
    • 방법 : sleep 100 &
    • &(and)를 쓰면된다.

  • 포그라운드, 백그라운드 명령어

    백그라운드로 실행 : sleep 100 &

    사용자 passwd에 관한 내용 백그라운드로 다 찾는다 : find / -name passwd >> out2 > out &

    백그라운드 작업 목록 보기 : jobs

    jobs 검색 후, -는 이전 작업, +는 최근 작업이다.

    작업번호가 지시하는 작업을 백그라운드 작업으로 전환 : bg [% 작업번호]

    Ctrl+Z = 멈춤, Ctrl+C = 종료 (kill 명령어에서 kill-2번을 담당하는 컨+C)

    작업 전환 (백그라운드 → 포그라운드) : fg [%작업번호]

    백그라운드는 kill 명령어로 죽일 수 있다 : kill -15 [PID], kill [%작업번호]

    로그아웃 후에도 백그라운드 작업 계속 실행하기 : nohup

작업 예약
  • 정해진 시간에 한 번만 수행하기 (at)
  • 정해진 시간에 반복 수행하기 (crontab)
  • 현재 작업 예약을 볼 때 명령어 : at -l (atq)
    • 슈퍼유저 일 땐, atq 명령으로 모든 사용자의 at 작업 목록을 확인할 수 있다.
  • 작업 예약을 지우고 싶을 때 : at -d (atrm)
  • at을 사용할 때, Ctrl + D를 누르면 빠져나올 수 있다.
  • (파일위치) /var/spool/cron/atjobs

    at의 파일 위치

  • at 명령 사용 제한하기
  • /etc/at.allow 파일이 있으면 이 파일 안에 있는 사용자만 at 명령 사용 가능 (화이트리스트)
  • /etc/at.deny 파일만 있으면 이 파일에 사용자가 없어야 at 명령 가능 (블랙리스트)
    • /etc/at.deny를 빈 파일로 두면 모든 사용자가 at 명령 사용 가능 (초기 상태)
  • 정해진 시간에 반복 실행 (crontab)
    • crontab 이용하여 정해진 시간에 반복실행
    • EDITOR를 vi로 설정하는 것 : EDITOR=vi
    • crontab 파일 생성하고, 편집하기 : crontab -e (edit)
      • 30 11 1 * * /bin/ls -l ~user1 > ~user1/cron.out
      • 해석 : 11시 30분에 1일 되는 모든 n월에, 모든 n요일에 bin/ls -l을 해주고, 그 결과를 (root)에 있는 user1 안에 cron.out으로 결과를 남겨놔라.
    • crontab파일내용을 확인하고( l ),삭제하기( r ) : crontab -l , -r
    • (파일위치) sudo ls -l /var/spool/cron/crontabs

      crontabs의 파일위치이다.
